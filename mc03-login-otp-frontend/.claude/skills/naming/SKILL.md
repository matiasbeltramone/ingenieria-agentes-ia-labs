# Naming

- Sustantivos para clases y módulos; verbos para funciones y métodos.
- Sin abreviaturas ni prefijos de tipo (nada de `I` de interfaz ni `Impl`).
- Un concepto, un nombre. Nombres del dominio, no genéricos técnicos (evitar
  `Manager`, `Helper`, `Util`).
- Sufijos permitidos cuando aportan: `UseCase`, `Port`, `Adapter`, `Repository`.
